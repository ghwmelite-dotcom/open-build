## Content plan

### The engine: one stream → a week of content
Every live build already flags 3–4 **clip moments** (see each build's guide on the hub). That's your raw material:
1. **Before the stream** — post the **vote** ("3 problems, you pick — voting closes [day]").
2. **During** — go live; drop "🔴 we're live" on X + the channel.
3. **After** — cut **3–4 vertical clips** from the flagged moments and release them across the week, each ending with **"Vote on the next build → link."**

### Per‑platform role
- **YouTube** — home base: full live **VODs** + **Shorts** (the vertical clips). One playlist per build. Thumbnail = `yt-thumbnail.png` style.
- **TikTok** — discovery: punchy **vertical clips**, the "aha" moments, fast hooks. This is where new viewers come from.
- **X** — conversation + funnel: clips, **build‑breakdown threads** ("here's the exact prompt"), vote teasers, live alerts.

### 10 post ideas (Ghana‑specific, replicable angle)
1. "I asked AI to map which parts of Accra flood — live. 😳" (flood‑map reveal)
2. "How much does MoMo *actually* take? I built a calculator in 10 min." (fee shock)
3. "Madina → Circle: what's the real trotro fare? We built the answer."
4. "You don't need to code. Watch a real tool get built with AI." (empowerment)
5. "Every tool we build, you can rebuild **for free** — here's the exact prompt." (the replicable hook)
6. "The community picked this. Here's what we shipped." (IKEA‑effect / credit)
7. "3 problems. You pick one. We build it live." (vote teaser)
8. "NSMQ season — we built a quiz trainer. Drill anywhere." (edtech)
9. "Built in Ghana, with AI, in one sitting." (speed + place)
10. "Behind the scenes: how one prompt becomes a working app."

### Hook formula
**Real problem → "can AI fix this live?" → the result.** Lead with the Ghanaian problem in the first 2 seconds; show the working result; end with the vote CTA.

### Cross‑posting
